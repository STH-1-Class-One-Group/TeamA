# 검색 및 추천 범위 기준

기준일: 2026-03-21  
기준 브랜치: `codex/production-deploy`

## 목적
- 레이아웃을 흔들지 않고 검색/추천의 최소 제품 범위를 먼저 고정합니다.
- 이후 UI를 붙일 때도 Worker와 타입 계약을 그대로 재사용할 수 있게 합니다.

## 검색 v1
- 대상: 장소, 사용자 코스
- 입력: 자유 텍스트 `q`
- 정렬: 이름/제목 일치 우선, 지역/카테고리/태그/본문 보조 점수
- 출력 제한:
  - 장소 최대 10개
  - 코스 최대 5개
- 비대상:
  - 댓글 전문 검색
  - 축제 전문 검색
  - AI 기반 의미 검색

### API
- `GET /api/discovery/search?q=...`

## 추천 v1
- 기준: 특정 장소 1개
- 방식:
  - 같은 카테고리 우선
  - 같은 권역 가점
  - 공통 vibe tag 가점
- 출력: 추천 장소 최대 6개
- 목적: 장소 상세/피드/코스에서 후속 동선 제안의 기반 데이터 제공

### API
- `GET /api/discovery/recommendations?placeId=...`

## 비기능 기준
- 기존 지도/피드/코스/마이 레이아웃은 변경하지 않음
- Worker에서 직접 계산하며, 추가 DB 스키마는 만들지 않음
- 결과는 짧은 배열로 제한해 응답 크기를 통제함

## 추후 확장
- 개인화 추천
- 검색어 자동완성
- 축제/피드 통합 검색
- 클릭/좋아요 기반 랭킹 보정
