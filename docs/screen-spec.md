# JamIssue 화면 설계 기준

기준일: 2026-03-18  
기준 브랜치: `codex/production-deploy`

이 문서는 PRD 방향과 현재 UI 구조를 맞추기 위한 화면 설계 기준입니다.

## 1. 전역 규칙

- 모바일 우선
- 기준 폭은 `390px`
- 한 화면에서 가장 중요한 행동은 하나만 강조
- 하단 탭은 항상 고정
- 기술 스택 문구는 사용자 화면에 노출하지 않음
- 긴 설명보다 상태와 버튼 문구로 안내

## 2. 네비게이션 구조

하단 탭은 4개입니다.

- `지도`
- `피드`
- `코스`
- `마이`

규칙:
- 각 탭은 별도 페이지처럼 보여야 함
- `지도` 탭만 지도와 바텀 드로워를 사용
- `피드 / 코스 / 마이`는 지도 배경을 공유하지 않음

## 3. 지도 탭 규칙

### 목적
- 장소와 축제를 빠르게 고르고, 스탬프/후기/코스로 이어가는 시작점

### 구조
- 상단 브랜드 카드
- 카테고리 필터 칩
- 장소/행사 개수 메타
- 중앙 지도 카드
- 하단 장소 상세 드로워 또는 축제 정보 드로워
- 하단 네비게이션

### 바텀 드로워 규칙
- 기본은 닫힘
- 마커 클릭 시 `partial`
- 위로 확장하면 `full`
- 아래로 내리면 닫힘
- 닫히면 `selectedPlaceId` 또는 `selectedFestivalId` 해제

### 장소 드로워 안에서 보여줄 것
- 장소명 / 구 / 요약
- 카테고리 뱃지
- 방문 횟수 / 오늘 스탬프 상태
- 후기 작성 가능 여부
- 장소별 후기 목록

### 축제 드로워 안에서 보여줄 것
- 축제명
- 개최장소
- 시작일 / 종료일
- 도로명 주소
- 홈페이지 링크
- 축제는 정보 제공만 하고 스탬프 로직과 섞지 않음

## 4. 피드 탭 규칙

### 목적
- 실제 방문 후기만 탐색하는 공간

### 구조
- 상단 제목
- 후기 카드 목록
- 장소로 이동 버튼
- 댓글 / 좋아요

### 제약
- 후기 작성은 지도 탭의 장소 드로워에서만 시작
- 피드 탭은 소비 중심

## 5. 코스 탭 규칙

### 목적
- 운영자 큐레이션과 사용자 생성 경로를 함께 보여주기

### 구조
- 정렬 칩 `좋아요순 / 최신순`
- 사용자 생성 경로 목록
- 기본 큐레이션 코스 목록

### 제약
- 사용자 생성 코스는 `travel_session` 기반
- 최소 2곳 이상
- 같은 세션으로 중복 발행 불가

## 6. 마이 탭 규칙

### 목적
- 계정 상태와 탐험 기록 요약

### 구조
- 프로필 카드
- 통계 2개
  - 방문한 고유 장소 수
  - 누적 스탬프 수
- 내부 탭
  - `얻은 스탬프`
  - `내가 쓴 피드`
  - `생성한 코스`

### 제약
- 로그인 전에는 로그인 유도만 노출
- 로그인 후에만 닉네임 수정과 개인 데이터 노출

## 7. 버튼 / 상태 규칙

- 스탬프 버튼
  - 로그아웃이면 비활성화 + 로그인 필요 안내
  - 반경 밖이면 비활성화 + 거리 안내
  - 반경 안이면 활성화
  - 오늘 이미 적립했으면 완료 상태

- 후기 작성 버튼
  - 당일 해당 장소 스탬프가 있을 때만 활성화
  - GPS 진입만으로는 열리지 않음

- 코스 발행 버튼
  - 공개 가능한 `travel_session` 이 있을 때만 활성화

## 8. QA 체크리스트

- `지도` 탭 외 화면에서 지도 배경이 보이지 않는가
- 지도 탭에서 상단 카드 / 필터 / 지도 / 드로워가 서로 가리지 않는가
- 바텀 드로워가 `닫힘 / partial / full` 로 자연스럽게 움직이는가
- 스탬프 버튼이 반경 밖에서 비활성화되는가
- 후기 작성이 `stamp_id` 없이 열리지 않는가
- 마이페이지 통계가 고유 장소 수 / 누적 스탬프 수로 분리되는가
